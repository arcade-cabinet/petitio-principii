# Internationalization (i18n)

## Supported Locales

| Code | Language | Direction |
|------|----------|-----------|
| `en` | English (canon) | LTR |
| `es` | Spanish | LTR |
| `fr` | French | LTR |
| `ar` | Arabic | RTL |

## Grammar-Slot Policy

This is the critical rule for contributors adding content:

**UI chrome localizes. Game prose stays English.**

### What localizes

- Button labels, error messages, banners, and navigation hints — anything that is *interface* rather than *content*.
- Room template *structure*: sentence patterns like "You enter a room of [noun]." → localized sentence pattern.
- Share card descriptive text.
- Settings labels and aria text.

### What stays English

- **Surrealist fragment corpus** (`src/content/generated/surrealist.json`): These are derived from public-domain English and French-in-English-translation surrealist literature (Breton, Éluard, Jarry). Their meaning is in the sound and rhythm of their specific words. Translating them would break the voice. They are quotations, not UI copy.
- **Tracery grammar templates** (`scripts/grammars/`): The game's rhetorical narration is generated from RiTa-enriched Tracery grammars. These grammars are authored in English and produce English prose. They are part of the game's art, not its interface.
- **Hint text** (`src/features/terminal/hints.ts`): Onboarding hints are written as diegetic fragments of the argument. They are part of the surrealist voice.
- **All text emitted by the reducer** (`src/engine/core/reducer.ts`): Movement blocks, trace responses, opening narration.

The player-language setting affects the *frame* around the game, not the game itself. A Spanish-speaking player sees "Iniciar Argumento" on the landing but plays entirely in English prose once they begin. This is a deliberate product decision: the game is about an English-language rhetorical tradition, and the incantation phrases are generated from English word lists.

## Implementation

### Setup

i18next is initialized in `src/lib/i18n.ts` before React mounts (imported in `main.tsx`). Language is detected in this order:

1. `?lng=` URL parameter
2. `localStorage` key `pp.lang`
3. Browser `navigator.language` (first two characters)
4. Fallback: `en`

### Adding a translation

1. Add a new file at `src/lib/locales/<code>.json` matching the structure of `en.json`.
2. Import it in `src/lib/i18n.ts` and add to `resources`.
3. Add the code to `SUPPORTED_LANGUAGES`.
4. If the language is RTL, add its code to the `RTL_LANGS` set in `i18n.ts`.

### RTL (T83)

Arabic (`ar`) triggers `dir="rtl"` on `<html>` via `applyDir()` in `i18n.ts`. The CSS layout uses logical properties throughout:

- `padding-inline-start` / `padding-inline-end` instead of `padding-left` / `padding-right`
- `margin-inline-start` / `margin-inline-end` instead of physical margins
- Tailwind's logical-property utilities (`ps-`, `pe-`, `ms-`, `me-`) preferred in new code

The CrystalField canvas backdrop and the argument map rail are direction-independent (absolute coordinates). The text zones and panel borders respond correctly to `dir="rtl"` automatically because they use logical properties.

### Language switcher

The landing screen exposes a compact language switcher (EN / ES / FR / AR buttons). Selecting a language persists to `localStorage` and updates the `lang` + `dir` attributes on `<html>` immediately via `i18next.on("languageChanged", applyDir)`.
